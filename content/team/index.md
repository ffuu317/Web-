## 每个人遇到的难点与解决办法

### 傅佳仪：虚拟机部署站点

| 难点 | 原因 | 解决办法 |
|---|---|---|
| 虚拟机中站点无法访问 | Nginx 未启动、端口未开放，或站点目录没有指向生成后的网页文件 | 先用 `systemctl status nginx` 检查服务状态，再确认 80 端口或自定义端口已放行，最后检查 Nginx `root` 是否指向包含 `index.html` 的目录 |
| 上传 `public/` 后页面出现 404 | 本地生成目录、上传目录和 Nginx 配置目录不一致 | 上传后在虚拟机中用 `ls` 检查文件位置，确保 `index.html`、`css/`、`images/` 等内容都在 Nginx 指定目录下 |
| SSH 远程管理和文件权限不稳定 | SSH 服务、登录用户、端口或 `/var/www/` 目录权限没有配置好 | 检查 `sshd` 服务和登录命令，部署目录用专门用户管理，必要时用 `chown` 或 `chmod` 调整站点目录权限 |

### 付陈悦：内容整理与 Markdown 编写

| 难点 | 原因 | 解决办法 |
|---|---|---|
| 实验过程内容层级混乱 | 原始资料来自聊天记录、截图和表格，直接复制会显得零散 | 按“实验目标、环境准备、操作过程、问题解决、运行验证”的顺序重新整理为 Markdown |
| 命令和代码片段不够清楚 | 普通文本中混着命令，阅读时不容易区分 | 将命令写成行内代码或代码块，例如 `hugo server -D`、`git status`，并补充每条命令的作用 |
| 图片或截图路径容易失效 | 使用了本地绝对路径，部署到网站后无法读取 | 把图片统一放到 `static/images/`，页面中使用 `/images/文件名.png` 的方式引用 |

### 李畅：页面样式和布局优化

| 难点 | 原因 | 解决办法 |
|---|---|---|
| 页面初始效果像默认网页 | 自定义 CSS 没有正确加载，或浏览器缓存了旧样式 | 在 `hugo.toml` 中配置 `custom_css`，确认 `/css/custom.css` 能访问，并用 `Ctrl + F5` 强制刷新 |
| 表格在手机端显示拥挤 | 分工表和问题表列数较多，小屏幕宽度不足 | 给表格增加横向滚动和更稳定的间距，避免内容被压缩变形 |
| 首页重点不突出 | 信息都堆在正文里，缺少视觉分组 | 用卡片、标题、间距和浅色背景区分“完成情况”“验收项目”“任务安排”等模块 |

### 冯美娟：网站栏目和链接检查

| 难点 | 原因 | 解决办法 |
|---|---|---|
| 菜单名称和页面内容不完全对应 | 原来页面更偏“分工”和“报告”，没有明显贴合评分表 | 将栏目调整为“实验记录、任务问题、部署、验收清单”，让每个页面承担一个明确功能 |
| 页面链接容易跳错 | Hugo 生成后的站内路径和 Markdown 中写的路径不统一 | 统一使用 `/posts/`、`/team/`、`/deploy/`、`/report/` 等站内路径，并逐个点击检查 |
| 验收材料不够集中 | 截图、命令、部署信息分散在不同页面 | 在“验收清单”中按评分项整理需要保留的材料，方便最后写报告时直接引用 |

### 夏涵欣：实验报告整理

| 难点 | 原因 | 解决办法 |
|---|---|---|
| 报告结构容易写散 | 网站搭建、运行环境、Linux 部署和远程管理内容较多 | 按评分表六项组织报告：项目配置、Web 运行环境、Linux 环境、账户配置、远程管理、站点部署 |
| 截图材料不完整 | 只截网页效果不够，还需要体现配置和运行过程 | 补充 GitHub 仓库、Hugo 本地运行、虚拟机部署、Nginx 配置、SSH 登录和最终访问效果截图 |
| 问题解决过程不好汇总 | 每个人遇到的问题分散在不同任务里 | 从本页提取各成员的典型问题和解决办法，写入报告的“遇到的问题与解决方案”部分 |

## 协作与提交规范

- 内容与图片素材统一放在 `static/images/`
- 实验过程文章放在 `content/posts/`
- 页面修改后先本地运行 `hugo server -D` 预览
- 提交信息尽量写清楚类型，例如 `docs:` 文档、`style:` 样式、`fix:` 修复
